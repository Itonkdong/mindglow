# Project Guidelines Index

Load only the guideline file(s) needed for the current task.

- `GENERAL_GUIDELINES.MD`: Always-on project rules that apply across the codebase.
- `ERROR_HANDLING_GUIDELINES.MD`: Failure-handling, exception design, and error-reporting rules.
- `SERVICE_GUIDELINES.MD`: Rules for orchestration modules, pipeline-style services, and domain workflow boundaries.

The current guideline set is intentionally centered on:
- modular experimentation,
- clear pipeline/service boundaries,
- explicit error handling,
- maintainable framework code.
